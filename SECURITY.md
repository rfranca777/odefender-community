# Security Policy — ODefender Community

## Reporting a Vulnerability

If you discover a security vulnerability in any ODefender Community project, please **DO NOT** open a public GitHub issue.

### Responsible Disclosure

1. **Email**: Send details to **rafael.franca@live.com** with subject: `[SECURITY] ODefender — <project> — <brief description>`
2. **Include**:
   - Affected project and version
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)
3. **Response**: Acknowledgment within **48 hours**
4. **Timeline**: Critical fixes within **7 days**

### Security Standards We Follow

- ✅ No credentials, tokens, or secrets in any codebase
- ✅ Template configuration files with placeholder values only
- ✅ `.gitignore` patterns exclude all sensitive files
- ✅ All projects support `reportOnly` mode for safe testing
- ✅ Signed commits recommended for all contributors
- ✅ Branch protection on `main` branch
- ✅ Dependency review for all pull requests

### Acknowledgments

Security researchers who responsibly disclose valid vulnerabilities will be credited in the relevant project's CHANGELOG (with permission).
